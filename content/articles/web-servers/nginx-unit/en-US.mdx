# NGINX Unit - Universal Dynamic Application Server

NGINX Unit is a lightweight, open-source application server that can run web applications written in multiple programming languages. It offers dynamic configuration and zero-downtime updates through a RESTful API.

## What is NGINX Unit?

NGINX Unit is a dynamic web and application server designed to run applications in multiple languages. Created by the NGINX team, it provides a unified solution for serving Python, PHP, Go, Perl, Ruby, JavaScript (Node.js), and Java applications.

## Key Features

- **Polyglot Support**: Run apps in Python, PHP, Go, Ruby, Perl, Node.js, Java
- **Dynamic Configuration**: Update config via REST API without restarts
- **Zero Downtime**: Apply changes without dropping connections
- **Process Management**: Automatic process spawning and lifecycle management
- **TLS Support**: Built-in HTTPS with SNI support
- **WebSocket**: Full WebSocket support
- **Lightweight**: Minimal resource footprint

## Installation

Installing NGINX Unit:

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install unit

# CentOS/RHEL
sudo yum install unit

# Install language modules
sudo apt install unit-dev unit-python3.11 unit-php unit-go
```

## Basic Configuration

Configure Unit via its REST API:

```bash
# View current configuration
curl http://localhost:8443/config

# Upload configuration
curl -X PUT --data-binary @config.json \
     --unix-socket /var/run/control.unit.sock \
     http://localhost/config
```

## Running a Python Application

Example configuration for a Python WSGI app:

```json
{
    "listeners": {
        "*:8080": {
            "pass": "applications/python_app"
        }
    },
    "applications": {
        "python_app": {
            "type": "python 3.11",
            "path": "/var/www/python_app",
            "module": "wsgi",
            "callable": "application"
        }
    }
}
```

## Running a PHP Application

Configuration for PHP applications:

```json
{
    "listeners": {
        "*:8080": {
            "pass": "applications/php_app"
        }
    },
    "applications": {
        "php_app": {
            "type": "php",
            "root": "/var/www/php_app",
            "index": "index.php"
        }
    }
}
```

## Running Node.js Applications

Configuration for Node.js:

```json
{
    "listeners": {
        "*:8080": {
            "pass": "applications/node_app"
        }
    },
    "applications": {
        "node_app": {
            "type": "external",
            "working_directory": "/var/www/node_app",
            "executable": "/usr/bin/node",
            "arguments": ["app.js"]
        }
    }
}
```

## Process Management

Configure process limits and scaling:

```json
{
    "applications": {
        "python_app": {
            "type": "python 3.11",
            "path": "/var/www/python_app",
            "module": "wsgi",
            "processes": {
                "max": 10,
                "spare": 2,
                "idle_timeout": 20
            }
        }
    }
}
```

## TLS Configuration

Enable HTTPS:

```json
{
    "certificates": {
        "example.com": {
            "key": "/path/to/key.pem",
            "chain": "/path/to/cert.pem"
        }
    },
    "listeners": {
        "*:443": {
            "pass": "applications/app",
            "tls": {
                "certificate": "example.com"
            }
        }
    }
}
```
